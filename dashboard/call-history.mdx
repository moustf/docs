# Call History & Sessions

The Call History feature provides a comprehensive searchable database of all your voice AI conversations. Search through thousands of calls, apply advanced filters, review full transcripts, and analyze individual session performance.

## Overview

Call History is your system's memory‚Äîevery conversation is logged, transcribed, and made searchable. Whether you need to review a specific call from last week or analyze patterns across thousands of sessions, Call History gives you the tools to find exactly what you need.

### Key Capabilities

- **Complete call archive**: Every conversation automatically saved
- **Advanced search**: Find calls by ID, agent, status, duration, and more
- **Multi-level filtering**: Combine filters for precise results
- **Sortable columns**: Sort by time, duration, or cost
- **Detailed transcripts**: Full conversation history with timestamps
- **Audio playback**: Listen to recorded calls
- **Performance metrics**: Per-call analytics
- **Bulk operations**: View and analyze multiple calls

<Info>
Call History is separate from the Live Calls feature. Live Calls shows active conversations from the last 10 minutes, while Call History contains your complete archive of all calls (active and completed).
</Info>

## Accessing Call History

The Call History feature is available as a dedicated page:

1. Navigate to **Call History** from the main navigation menu
2. The call history table loads with your most recent calls
3. Apply filters and search as needed

```
Main Menu > Call History
```

**Default view**:

- Shows calls with status: IN_PROGRESS and COMPLETED
- Sorted by time (newest first)
- Displays 10 calls per page
- All agents included

## Understanding Call Statuses

Every call in the system has one of five possible statuses:

### Status Types

**PENDING**

- Call has been initiated but not yet started
- Waiting for connection or agent availability
- Color: Yellow badge

```
Example: Outbound call queued but not yet dialing
```

**IN_PROGRESS**

- Call is currently active
- Conversation is ongoing
- Color: Blue badge with pulsing indicator

```
Example: Customer is talking to AI agent right now
```

**COMPLETED**

- Call finished successfully
- Full transcript and audio available
- Color: Green badge

```
Example: Normal call that ended gracefully
```

**NO_ANSWER**

- Call was initiated but not answered
- No conversation took place
- Color: Gray badge

```
Example: Outbound call rang but nobody picked up
```

**FAILED**

- Call encountered an error and couldn't complete
- May have technical issues
- Color: Red badge

```
Example: Connection dropped, system error, invalid number
```

<Warning>
By default, the table shows only IN_PROGRESS and COMPLETED calls. To see failed or unanswered calls, modify the status filter.
</Warning>

## Call History Table

The main table displays your calls in an organized, sortable format:

### Table Columns

**Time**

- When the call started
- Format: "MM/DD/YYYY, HH:MM:SS AM/PM"
- Timezone: Your browser's local timezone
- Sortable: Click column header to sort

```
Example: 11/16/2024, 2:34:15 PM
```

**Agent**

- Name of the voice AI agent that handled the call
- Click agent name to filter table to only that agent
- Sortable: Alphabetically

```
Example: Customer Support Agent
```

**Status**

- Current call status (see statuses above)
- Color-coded badges for quick recognition
- Filterable: Use status filter dropdown

**Duration**

- How long the call lasted
- Format: "Xm Ys" (minutes and seconds)
- Sortable: Click column header to sort by duration

```
Examples:
- 2m 34s (2 minutes, 34 seconds)
- 45s (45 seconds)
- 12m 8s (12 minutes, 8 seconds)
```

**Channel**

- Communication channel used for the call
- Options: Phone, Web, WhatsApp
- Icon + text label

**Cost**

- Total cost of the call (if applicable)
- Currency and amount
- Sortable: Click column header to sort by cost

```
Example: $0.12
```

**Actions**

- View details button (eye icon)
- Opens call details drawer

<Tip>
Click any column header to sort by that column. Click again to reverse the sort order (ascending/descending).
</Tip>

## Filtering and Search

Powerful filtering capabilities help you narrow down thousands of calls to find exactly what you need.

### Available Filters

The toolbar at the top of the table provides multiple filter options:

#### Status Filter

Filter calls by their current status:

**How to use**:

1. Click the **Status** dropdown button
2. Check/uncheck status options:
   - ‚úÖ In Progress
   - ‚úÖ Completed
   - ‚òê Pending
   - ‚òê Failed
   - ‚òê No Answer
3. Click outside to apply filter

**Example use cases**:

```
Find failed calls only:
- Uncheck all except "Failed"
- Review error patterns

Find completed calls only:
- Uncheck all except "Completed"
- Analyze successful conversations

Find active calls:
- Uncheck all except "In Progress"
- Similar to Live Calls tab
```

<Note>
By default, only "In Progress" and "Completed" are checked. This hides failed and unanswered calls from the initial view.
</Note>

#### Channel Type Filter

Filter by communication channel:

**Options**:

- üìû Phone (Telephone)
- üåê Web
- üì± WhatsApp

**How to use**:

1. Click the **Channel** dropdown
2. Select one or more channels
3. Click outside to apply

```
Example: Phone support analysis
- Select only "Phone"
- Review phone-specific patterns
- Compare to Web channel performance
```

#### Agent Filter

Filter by specific voice AI agents:

**How to use**:

1. Click the **Agent** dropdown or agent name filter field
2. Type agent name or select from list
3. Press Enter or click outside to apply

```
Example: Compare two agents
1. Filter to "Agent A"
2. Note average duration and success rate
3. Clear filter
4. Filter to "Agent B"
5. Compare metrics
```

<Info>
Agent filtering in Call History works the same as the dashboard agent filter‚Äîit's synchronized across the application for consistency.
</Info>

#### Call ID Search

Search for a specific call by its unique identifier:

**How to use**:

1. Locate the search box (typically in the filter toolbar)
2. Enter the full or partial call ID
3. Press Enter or click search

```
Example session ID: abc-123-def-456

Search: "abc-123" ‚Üí Finds the matching call
```

**When to use**:

- Customer reports an issue and provides their session ID
- Following up on a specific call from logs
- Debugging a particular session

#### Duration Filter

Filter calls by how long they lasted:

**Conditions available**:

- Is between (range)
- Is greater than (minimum)
- Is less than (maximum)
- Is equal to (exact)

**How to use**:

1. Click the **Duration** filter button
2. Select condition type
3. Enter value(s) in seconds
4. Click Apply

**Examples**:

```
Find very short calls (< 30 seconds):
- Condition: Is less than
- Value: 30
- Use case: Identify connection issues

Find long calls (> 5 minutes):
- Condition: Is greater than
- Value: 300
- Use case: Find complex conversations

Find calls in specific range:
- Condition: Is between
- First value: 60
- Second value: 180
- Use case: Standard call length (1-3 minutes)
```

<Tip>
Combine duration filtering with status filtering to find patterns like "short completed calls" (successful quick resolutions) vs "short failed calls" (connection issues).
</Tip>

## Troubleshooting

### Calls Not Appearing

**Problem**: Expected calls don't show in the table

**Check**:

1. **Status filter**: Ensure the call's status is checked

   - Example: Looking for a failed call but only "Completed" is checked

2. **Date range**: Verify date range includes when the call occurred

   - Expand to "This Month" or "Custom" with wider range

3. **Agent filter**: Confirm correct agent is selected or use "All agents"

4. **Pagination**: Call may be on a different page

   - Check total count at bottom
   - Navigate through pages

5. **Search/filters**: Clear all filters and try again
   - Click "Reset" or "Clear Filters"

### Can't Play Audio Recording

**Problem**: Audio player shows error or doesn't load

**Solutions**:

1. **Call still in progress**: Audio only available for completed calls

   - Wait for call to complete

2. **Browser compatibility**: Use Chrome, Firefox, or Edge

   - Some browsers have limited audio codec support

3. **Recording not available**: Some calls may not have recordings

   - Check call status and metadata
   - Verify recording was enabled for this session

4. **Network issue**: Slow connection may timeout
   - Refresh page and try again
   - Check internet connection

### Transcript Missing or Incomplete

**Problem**: Conversation tab shows no transcript or partial transcript

**Possible causes**:

1. **Call in progress**: Transcript builds as call proceeds

   - Refresh to see latest messages

2. **ASR failure**: Speech recognition failed for some segments

   - Check Overview tab for ASR errors
   - May see gaps in transcript

3. **Data not synced**: Brief delay in transcript availability

   - Wait 30-60 seconds after call ends
   - Refresh drawer

4. **Audio quality issues**: Poor connection led to transcription failure
   - Review audio quality in recording
   - Check ASR processing times in Overview

### Slow Table Loading

**Problem**: Call history table takes long time to load

**Solutions**:

1. **Reduce date range**: Narrow from "This Month" to "This Week"
2. **Apply filters**: Load fewer calls by filtering
3. **Reduce page size**: Use 10 or 25 instead of 100 per page
4. **Check network**: Slow connection affects load time
5. **Clear browser cache**: Old cached data may cause issues

<Tip>
For best performance, avoid loading hundreds of calls at once. Use filters to narrow results, then paginate through as needed.
</Tip>

### Column Sorting Not Working

**Problem**: Clicking column headers doesn't sort

**Check**:

1. **Look for sort indicator**: Arrow icon should appear in column header
2. **Ensure column is sortable**: Not all columns may support sorting
3. **Refresh page**: Browser state may be stale
4. **Try different column**: Verify sorting works at all
